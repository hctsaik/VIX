在 FiftyOne App 裡覆核

把 VIX 的訊號丟進視覺化 App,用滑鼠點選覆核(真實操作截圖)。

Tier B 喜歡用滑鼠點、用眼睛看?把 VIX 的訊號丟進 FiftyOne App,在 @vix/review 外掛裡點選覆核。需 Tier-2(Python 3.11 + FiftyOne)。下面的截圖都是在真實的 pothole 資料上實際操作。

開啟 App

vix diagnose ./dataset --labels voc --weights best.pt --audit
vix weakness-report --worklist        # 把待辦轉成可點的 saved views(vixq:* 標籤)
vix app                               # 開 FiftyOne + @vix/review 外掛
成功的樣子:瀏覽器自動開啟 http://localhost:5151,看到影像網格;左側出現 review/pass 與 vixq:* 工作清單的 saved views;右上可開 operator 瀏覽器(快捷鍵 `)。
App 開不起來 / 空白?九成是 Python 版本:FiftyOne 需要 Python 3.11。請在 .venv311 裡跑(見 安裝),不要用基礎的 3.13/3.14。其次檢查 python -c "import fiftyone" 是否成功。

實際操作(逐步截圖)

  1. 用真實資料開啟 App 的網格檢視。
  2. 打開 operator 瀏覽器,選 generate_weakness_report 並挑 eval。
  3. 在面板裡直接看弱點報告(mAP、per-class、混淆)。
  4. 用 flag_label_issues / audit_label_errors 標出疑似錯標,形成工作清單。
  5. 點一張被標記的樣本,看到那個有問題的框。
  6. 用 flag_loose_boxes(SAM)找出太鬆的框。
完整 10 步走查(含可點的覆核佇列):見 GUI_WALKTHROUGH.html。

逐張覆核你的資料:覆核佇列

一句話:你先挑幾張「我確定標對了」的圖,VIX 就把「最不像這些圖」的排到最前面讓你先看 —— 一張圖越不像你的正確範例,越可能有問題,越該優先檢查。

會用到縮圖牆上方工具列那排 VIX 按鈕(橘色)。若視窗太窄、工具列空間不夠,它們會收進工具列上的橘色 ⌄ 選單裡:

按鈕作用
✓ 確認正確樣本(golden)告訴 VIX「這幾張我檢查過、標註是對的」,當成比對用的正確範例(VIX 內部叫 golden)。
⊘ 標記誤報排除這張其實沒問題、是系統虛驚一場 → 按它把這張移出清單、之後不再出現。
🔄 重新整理佇列橘色實心按鈕。每次你確認了正確範例或排除了誤報,按它一下用最新資訊重排清單。
  1. 挑幾張正確的圖:在 Samples 分頁勾選 5–10 張你確定標註正確、長相不同的圖。
  2. 按 ✓ 確認正確樣本(golden):工具列點這顆 → 跳出的小視窗按 Execute(執行)。這幾張就成了比對基準。
  3. 開啟覆核清單:在分頁列最右邊按 + → 選 VIX: 覆核佇列。若它顯示「⚠ 覆核佇列尚未就緒」,代表你還沒確認任何正確範例 —— 回去做步驟 1、2。
  4. 按橘色的 🔄 重新整理佇列。
  5. 逐張處理:最上面是最該先看的圖(「風險」欄只是排序分數,高 = 最不像你的正確範例、該優先看,不代表它一定錯)。每列可:看圖(跳到該圖)、確認→golden(其實是對的 → 收進正確範例)、誤報排除(虛驚一場 → 移出)。
  6. 處理完一列它就消失;重複到清空。你確認的正確範例越多,排序越準。
誠實提醒:
  • 沒有任何正確範例(golden)時,清單停用並顯示「尚未就緒」—— VIX 不會假裝有把握。
  • 排序是「和正確範例多不像 + 模型信心多低 + 疑似標錯」的綜合,決定看的順序,不是「這張一定錯」的判決。
  • 若你載入的校準是別的資料集算出來的,清單也會顯示「尚未就緒」—— 刻意如此,避免用錯基準排序。
