# VIX v0.1 技術規格 —「可見性」

> 本文件把 [discussion/consensus.md](../discussion/consensus.md) 的決策,補成「拿著能開工」的實作級規格。範圍僅 v0.1。所有決定可回溯至 Round 1–4 討論紀錄。

## 0. 範圍與前提

- **目標**:單人地端、air-gapped、無 Enterprise、無 API、無自建 plugin。解決首要痛點:「這批新資料加進去後,模型哪些預測變奇怪了?」
- **已確認約束**:保密/法規、資料不外流、無預算。**SSO / 法庭級稽核 v0.1 不需要**(使用者裁決)。
- **架構鐵則**:`vix/core/` **不得 import 任何 fiftyone**;只有 `vix/adapters/` 碰 FiftyOne。FiftyOne 是「可重建的工作空間 + 視覺化/運算引擎」,**不是事實來源**。

## 4. OutlierScorer 公式(雙軸,刻意不混合)

> **決策:不把 YOLO 信心與 DINO 距離加權成單一分數。** 保持兩軸獨立 → (a) 避免任意加權;(b) 每個 flag 都有人類可讀的理由,直接服務「可解釋性」初衷。

對影像中每個預測偵測 `i`(預測類別 `c`、裁切後 DINOv2 embedding `e_i`):

- **軸 1 — 信心**:`conf_i = YOLO confidence`。
- **軸 2 — 新穎度**:`knn_dist_i = mean cosine distance(e_i, 該影像所屬 golden-set 中類別 c 的 k 個最近鄰)`,`k=10`(類別樣本 < k 時用全部並標 `low_support`)。距離在**原始 DINOv2 空間**算(非 UMAP)。

影像級彙總:`yolo_conf_max = max_i conf_i`;`knn_dist = max_i knn_dist_i`(最離群的偵測主導覆核)。

---

## 5. ThresholdPolicy(per-class 百分位)

- 對每個類別 `c`,從**golden set**(訓練分布)計算:
  - `conf_thr[c]` = golden 中類別 c 正確偵測之 conf 的第 **5** 百分位 → `conf_i < conf_thr[c]` ⇒ flag `low_conf`。
  - `dist_thr[c]` = golden 中類別 c 之類內 kNN 距離的第 **95** 百分位 → `knn_dist_i > dist_thr[c]` ⇒ flag `far_from_known`。
- **路由**:任一 flag 成立 ⇒ tag `review` 並寫 `flag_reason`;兩軸皆過 ⇒ tag `pass`。
- **儲存**:`thresholds.json`(per-class 兩值 + 參考 golden 快照 id + 日期 + 百分位參數),**DVC 追蹤**。
- **誰算/何時**:`vix calibrate`(或 `vix guard` 首次執行)計算;golden set 實質變動時重算。README 寫明。

---

## 6. FrozenReference(v0.1 = embedding-based,零訓練)

- **anchor set**:從初始標註抽 5–10%,tag `anchor`,**永不進訓練、永不參與門檻計算**。
- **凍結 baseline**:pin DINOv2 權重 + 把 anchor 各類 embedding 存成 `anchor_ref.npy`(即 A 條件的「凍結 embedding snapshot」,一物兩用:當 v0.2 KS 基準)。
- **每批 guard 訊號**(`vix guard`):
  1. **類別漂移**:新批次各類 centroid(凍結 DINOv2 空間)vs anchor centroid 的 cosine 位移。
  2. **標籤一致率**:anchor 樣本中,其 golden kNN 多數類別 == 自身標籤 的比例。
- **單人自我 gate**:位移 > 門檻 或 一致率較基準下降 > **5%** ⇒ `vix guard` 寫一筆 `guard_alert` 進 DecisionLog 並要求 `--ack "理由"` 才放行(單人書面自審,非雙簽)。
- 完整版(凍結 reference YOLO + 預測分布 KL gate)→ v0.2。

---

## 7. DecisionLog(append-only)

- **格式**:JSONL,以 `open(path, "a")` 追加,**永不改寫既有行**。
- **每行**:`ts_utc`(UTC,系統時鐘)、`event`(route/review/guard_alert/export)、`vix_hash`、`batch_id`、`reviewer_id`(預設 `"auto"`/`$USER`)、`decision`、`scores`({conf, knn_dist})、`thr_version`、`prev_hash`。
- **選用 hash-chain**:`prev_hash = sha256(上一行序列化)`,逐行串接 → 近乎零成本的防竄改證據(回應稍早「不可竄改稽核」顧慮,但**不強制**)。

---

## 8. Manifest(DVC 追蹤對象)

- `manifest.jsonl`,每行:`vix_hash`、`src_path`(相對)、`batch_id`、`ingested_at`、`label_version`、`tags`。
- 這是 ingest 重建 FiftyOne Dataset 的依據,也是 DVC 版本邊界。

## 9. DVC 策略(關鍵:版控什麼 vs 重建什麼)

| DVC + Git 追蹤(=事實來源) | 不追蹤(可重建/拋棄) |
|---|---|
| 影像目錄、`manifest.jsonl`、`thresholds.json`、`anchor_ref.npy`、`decision_log.jsonl`、匯出的 golden 標籤 | **MongoDB 內的 FiftyOne 狀態**(由 `vix ingest` 從 manifest+媒體重建) |

→ FiftyOne 的 Mongo 狀態是**派生物**;`git`+`dvc` commit 即一個可重現的資料快照(commit 可 GPG 簽章)。這填補了 OSS 無 snapshot 的缺口。

## 10. FiftyOneAdapter 介面(唯一膠水層)

```python
class FiftyOneAdapter:
    def sync_dataset(self, manifest: Manifest) -> "fo.Dataset": ...      # 從 manifest+媒體重建,不寫業務欄位
    def compute_embeddings(self, ds, model_key: str) -> np.ndarray: ...  # 回 numpy,與 fo 解耦
    def build_knn_index(self, ds, embeddings_field: str) -> str: ...     # 包 compute_similarity(backend="lancedb"),回 brain_key
    def attach_fields(self, ds, hash2vals: dict) -> None: ...            # 把 VIX 分數寫成唯讀 field
    def apply_tags(self, ds, hash2tags: dict) -> None: ...
    def pull_review_decisions(self, ds) -> list[Decision]: ...           # 讀 App 內人工 tag → VIX Decision
    def launch_app(self, ds, saved_views: dict) -> None: ...             # 開 App + 預設 review/pass views
```
`vix/core/*` 只認 `np.ndarray` / `dict` / `Decision`,從不 import fiftyone → 抽換只改本檔。

## 11. CLI 指令(8 個)

| 指令 | 輸入 | 動作 | 輸出 |
|---|---|---|---|
| `vix ingest <folder> --batch <id>` | 資料夾 | 算 hash、寫 manifest、adapter 重建 Dataset | manifest 行、Dataset |
| `vix infer --weights <pt>` | YOLO 權重 | 推論 → `yolo_detections`/`yolo_conf_max` | fields |
| `vix embed` | — | DINOv2 逐 bbox embedding + 建 kNN index | `anchor_ref.npy`、brain_key |
| `vix calibrate` | golden set | 算 per-class 百分位門檻 | `thresholds.json` |
| `vix route` | thresholds | 雙軸判定 → tag + `flag_reason` + DecisionLog | tags、log |
| `vix guard [--ack ".."]` | anchor_ref | 漂移/一致率訊號 + 自我 gate | `guard_alert` log |
| `vix review` | — | (= `vix app` 別名)開覆核視圖 | — |
| `vix export <dst>` | golden tag | **單向**匯出 → YOLO txt + data.yaml | 訓練可直接吃的資料夾 |
| `vix app` | — | 開 FiftyOne App + 預設 SavedViews | localhost:5151 |

## 12. 驗收標準(v0.1 = Done)

- **功能**:在一台 air-gapped 機器上,能完成 `ingest → infer → embed → calibrate → route → guard → app(人工覆核)→ export` 全鏈,匯出檔可被既有 YOLO 腳本零改動訓練。
- **可解釋**:每張被 `review` 的圖都有 `flag_reason`(`low_conf` 或 `far_from_known`),人能一眼知道為何被攔。
- **採用面**(consensus §8):決策時間 < 30 分/批;連續 4 週使用;主動排除張數 > 0。
